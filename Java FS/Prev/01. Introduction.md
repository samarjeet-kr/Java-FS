### Concept of dependency
- To understand dependency injection we first have to look at what a dependency actually is in the world of programming.
- In simple terms if class A uses a method or property of class B, then class A is dependent on class B.

```java
public class Car{
	private Engine engine;
	public car(){
		// The car is tightly coupled to a specific  Engine
		this.engine = new petrolEngine();
	}
	public void drive(){
		engine.start();
	}
}
```

### Types of dependency injection in Spring
- Spring gives 3 main ways to inject dependencies.

1. *Constructor Injection*: Dependencies are provided through the class constructor.  this is the Gold Standard because is allows for immutable.
	
```java
@Component
public class MyService{
      private final MyRepository repo;

      public MySerice(MyRepository repo){
          this.repo = repo;
      }
}
```

2. *Setter Injection* :  Dependencies are provide through public setter method. This is useful for optional dependencies that might change later.

```java
@Autowired 
public void setRepo(MyRepository repo){
         this repo = repo;
}
```

3. *Field Injection* : Using `@Autowired` directly on the variable. Its the shortest code but generally discouraged because it makes harder to test the class without the spring container.

```java
@RestConstroller
public class UserController {
	@Autowired
	private EmailService emailService; // This is field injection.
	
	@GetMapping("/notify")
	public String notifyUser(){
		emailService.sendEmail("Welcome to SpringBoot !");
		return "Notification Sent";
	}
}
```

### Spring IoC (Inversion of control)

### What is inversion of control ? 
IoC is a design principle where the controller object or portions of  a program is transferred to a container or a  framework.
#### Traditional Flow
```java
Engine e = new Engine();
```
#### IoC Flow
The framework calls your code it creates the object and gives it to you when you need it.
#### The core Mechanism 
- Dependency injection IoC is the concept but dependency injection is the process.
- Where the container injects dependencies into a BEAN at runtime. 
- Think of IoC as department of labour and dependency injection is the recruiter. 
- Who actually places the right worker in the right office.

### How Spring IoC works ? (Process)
1. *Metadata gathering* : Spring looks at your configuration ('Annotations' like `@Component` for `.xml` files).
2. *Bean definition* :  Spring creates a plan for every object it needs to manage.

Coding example
##### Steps : 
1.
```java
import org.springframework.streotype.Component;
@Component
public class EmailService{
	public void send(String message){
		System.out.println("Sending Email : " + message);
	}
}
```
2.
```java
@Component
public class NotificationManager{
	private final EmailService emailService;
	
	// IoC Container injects EmailService here automatically
	public NotificationManager(EmailService emailService){
		this.emailService = emailService;
	}
	public void notifyUser(String msg){
		emailService.send(msg);
	}
}
```

### Lombok
Lombok is the shorthand boiler plate code.
- Repetitive task like Getters, Setter, ToString and constructor.
- Lombok is a library that plugs into your ID and build tool to generate this code automatically through compilation. 
- To use this you simplify `@Annotation` to your class and `Lombok` does the heavy lifting behind the scenes. 
- The big 3 for data classes these are the most common Annotations used to clean-up models or entity classes.

- `@Getter` and `@Setter` generates `getFieldName()` and `setFieldName()` for every field `@ToString()`.

Syntax :
```java
User(id = 1, name = "John");
```

#### Constructor Annotations: 
- Creating constructs manually is tedious especially 3 fields.

	- (`@NoArgsConstructor` : It generates a constructor with no parameter (often required by JPA/Hibernate ) 

	- `@AllArgsConstructor` Generated a constructor with on parameter with every field int the classes.

	- `@RequiredArgsConstructor` : Generated a constructor for all final fields and fields marked with `@NonNull`.

Code Example : 
```java
@Builder
@Getter
public class User{
	private final int id;
	private final String nname;
	private final String email;
}

// Usage
User myUser = User.builder()
					.id(101)
					.name("Gemeni")
					.email("gemeni@example.com")
					.build();
```
